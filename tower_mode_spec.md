# タワーモード（TOWER MODE）仕様まとめと改善案

タワーモードは、プレイヤーがマップを進みながら敵を倒し、スキルやバフを集めながら最終的にタワーの最上階を目指す（またはどこまで到達できるかを競う）ローグライク形式のゲームモードです。

## 1. 全体フローと基本仕様

### マップ進行
- 各階層（Floor）ごとに複数のノード（部屋）から進む先を1つ選択します。
- 一度選択したノードへ進むと、マップ画面が隠れ、そのノードのイベントや戦闘が発生します。
- ノードでの事象が完了（敵を倒す、イベントの選択肢を選ぶ等）すると、次の階層へ進み、再びマップ画面が表示されます。

### ノードの種類
タワーモードには以下の種類のノードが存在します：
- **BATTLE / WEAK_BATTLE / ELITE / BOSS**: 敵との戦闘。勝利すると報酬（ゴールドやスコア等）が得られ、次の階へ。
- **TREASURE**: 宝箱部屋。3つの報酬から1つを選択して獲得します（装備、HP回復、大金など）。
- **SHOP**: 商店部屋。所持金（GOLD）を消費して、スキル、HP回復アイテム、パッシブバフなどを購入できます。割引ショップ（DISCOUNT_SHOPイベント）も存在します。
- **EVENT_SAFE**: 安全なランダムイベント。プレイヤーに有利な選択肢（バフ付与、ゴールド獲得など）が発生します。
- **EVENT_RISK**: 危険なランダムイベント。リスク（HP減少や所持金没収）を伴うが、大きなリターン（強力なスキルなど）が得られるギャンブル要素があります。

### リソース管理
- **HP**: プレイヤーの体力。0になるとゲームオーバー（タワー探索失敗）。
- **GOLD**: ショップや一部のイベントで使用する通貨。戦闘やイベントで獲得できます。
- **スキル（Skill）**: プレイヤーを強化する能力。イベントやショップで獲得・強化できます。

## 2. 改善点・拡張案（アップデート提案）

現在の仕様をベースに、より面白く、リプレイ性を高めるための改善案を提案します。

### UI / UX の改善
1. **リソースの常時確認機能**
   - *現状*: マップやイベント画面で現在の「所持ゴールド」や「現在のHP」が確認しづらい場面があります。
   - *改善案*: 画面上部に常に現在のHP、最大HP、所持ゴールド、所持スキルを表示する専用のヘッダーバー（HUD）を追加する。
2. **マップ進行度の可視化**
   - *現状*: 現在何階にいるかはテキスト表示されていますが、全体像（ボスの位置など）が分かりにくいです。
   - *改善案*: `Slay the Spire` のような、分岐のある全体マップツリーを可視化し、先のルートをある程度予測しながら進めるようにする。

### ゲームバランスとリプレイ性の向上
1. **レリック（恒久パッシブアイテム）の実装**
   - *現状*: バフはイベントで付与されますが、種類が限定的です。
   - *改善案*: 所持しているだけで永続効果を発揮する「アーティファクト（レリック）」システムを導入し、宝箱やボス討伐報酬としてドロップさせる。これにより、プレイごとの「ビルド（育成方針）」に多様性が生まれます。
2. **イベントのバリエーションと「連続イベント」**
   - *現状*: イベントは1回の選択で完結します。
   - *改善案*: 「特定のアイテムを持っていると特別な選択肢が出る」「前の階で助けたNPCが後の階で助けてくれる」といった、条件分岐や連続性のあるイベントを追加する。

### リスクとリターンの強調
1. **エリートノードのハイリスク化**
   - *改善案*: エリートノードをさらに強力にする代わりに、倒した際に必ず専用の強力な報酬（レアスキルや大金）を確定ドロップするようにし、プレイヤーが自らリスクを取りに行く動機を強める。
2. **呪いシステム**
   - *改善案*: 強力な恩恵を得る代わりに「呪い（デバフ）」を受けるイベントを追加し、ショップで高いお金を払って呪いを解く、といったリソース管理の駆け引きを生み出す。

## 3. 手動デバッグ機能について
今回のテスト実装にあわせ、**マップ画面の右上に小さな「DEBUG」ボタンを追加しました。**
このボタンを押すと、「GOLD」「SHOP」「TREASURE」「TRIAL」など、任意のイベントIDを入力して強制的にそのイベントやノードを発生させることができます。プレイアビリティやバランス調整の確認にご活用ください！
